------------------------------------------------------------

Jenis media:
- image
- video
- favicon
- dokumen/pdf

Field metadata media yang direkomendasikan:
- id
- file_name
- original_name
- mime_type
- file_size
- storage_path
- media_type
- title
- alt_text
- caption
- category
- tags
- is_active
- created_at
- updated_at

Fitur penting:
- multi-upload
- thumbnail preview
- search
- filter by type
- safe delete prevention
- default fallback asset

Kebijakan delete:
- media yang sedang dipakai section lain tidak boleh dihapus langsung
- paling aman: soft validation sebelum delete

Kebijakan optimasi:
- image optimization sangat direkomendasikan
- jika memungkinkan:
  - buat versi web-optimized
  - pertimbangkan WebP
  - lazy loading on frontend
- tetapi implementasi final menyesuaikan kesiapan pipeline

Catatan:
- user belum ingin ukuran media dipatok keras
- maka pendekatannya:
  - beri rekomendasi
  - optimasi otomatis bila memungkinkan
  - hindari aturan yang terlalu kaku di tahap awal

16. STRATEGI VERSIONING DAN PUBLISHING
------------------------------------------------------------
Tujuan:
- aman saat edit
- mudah rollback
- mudah preview
- mengurangi risiko kehilangan perubahan

State yang direkomendasikan:
- draft
- saved
- previewable
- published
- archived snapshot (optional internal concept)

Workflow:
1. Admin edit draft
2. Admin save draft
3. Admin preview full page
4. Admin publish
5. Sistem membuat snapshot homepage
6. Admin bisa lihat riwayat versi
7. Admin bisa restore/publish ulang versi lama

Pendekatan teknis terbaik:
- konten aktif tetap disimpan di tabel normal
- saat publish, sistem membuat snapshot homepage lengkap (disarankan format JSON)
- snapshot menyimpan state final seluruh homepage

Isi snapshot ideal:
- version_number
- title/label optional
- full homepage config JSON
- published_by
- published_at
- notes optional

Keuntungan snapshot total homepage:
- restore cepat
- aman
- cocok untuk one-page semi-flexible
- lebih mudah dibanding versioning granular semua tabel secara kompleks

Autosave:
- boleh disiapkan untuk draft editing
- cukup sederhana di fase awal

18. STRATEGI ANIMASI DAN PARALLAX
------------------------------------------------------------
Prinsip:
- elegant motion
- subtle but noticeable
- premium feel
- performance-safe
- no heavy over-animation

Distribusi animasi:
A. Hero
- parallax utama
- overlay/gradient movement ringan
- subtle depth
- clean reveal for text

B. About
- reveal on scroll
- optional light depth effect

C. Video Highlight
- fade/slide reveal ringan

D. Services
- reveal on scroll
- hover effect ringan
- modal transition smooth

E. Gallery
- stagger reveal
- hover ringan
- smooth lightbox
- no heavy parallax

F. Footer / Location
- minimal motion only

Batas aman:
- maksimal 1 parallax utama
- maksimal 2–3 section dengan premium scroll effect ringan
- sisanya animasi biasa

Fallback low-spec:
- reduce motion on low-performance devices
- simplify on mobile
- respect prefers-reduced-motion
- disable heavy effects if necessary

Kontrol animasi di admin:
- jangan terlalu granular
- cukup:
  - enable/disable parallax hero
  - intensity ringan/sedang
  - overlay level terang/sedang/gelap
  - section animation on/off where relevant

------------------------------------------------------------
Target:
- tetap premium
- tetap cepat
- tetap aman di device low-spec

Karena paket hosting belum pasti:
- hindari desain backend yang terlalu berat
- optimasi media jadi penting
- video perlu perhatian khusus
- jangan mengandalkan resource server tinggi

Yang harus dijaga:
- upload workflow efisien
- query tidak boros
- aset frontend teroptimasi
- animasi tidak berat
- fallback aman jika shared hosting

Arsitektur sebaiknya:
- tetap mudah dipindahkan ke environment yang lebih tinggi di masa depan
- tidak terlalu mengikat ke solusi yang sulit dipindah

28. RISIKO PROYEK YANG SUDAH TERIDENTIFIKASI
------------------------------------------------------------
1. Scope creep dari semi-flexible menjadi full custom builder
Mitigasi:
- kunci batas composer dari awal

2. Video terlalu besar untuk hosting
Mitigasi:
- struktur fleksibel
- putuskan limit nanti
- optimasi dan fallback

3. Terlalu banyak animasi
Mitigasi:
- maksimal 1 parallax utama
- section lain ringan

4. Operator bingung dengan admin
Mitigasi:
- UI admin visual
- kontrol sederhana
- terminology mudah

5. Media berantakan
Mitigasi:
- media library terpusat
- metadata
- safe delete

6. Perubahan homepage berisiko merusak live site
Mitigasi:
- draft
- preview
- snapshot
- restore

31. RINGKASAN TEKNIS SANGAT SINGKAT
------------------------------------------------------------
Bangun website one-page Balkondes Bumiharjo di CI4 terbaru dengan visual mengikuti referensi. Fokus utama adalah branding, promosi, dan booking. Hero tunggal dengan parallax ringan menjadi pusat visual. CTA WhatsApp langsung memakai template chat otomatis, sedangkan booking/OTA memakai mini-linktree internal. Services bersifat dinamis dan dibuka dalam modal popup, bukan halaman detail. Gallery menampilkan 6 item awal dan memakai lightbox. Tidak ada form website. Admin hanya satu superadmin dengan sistem semi-flexible: ada Homepage Composer, Media Library, Services, Gallery, Booking Links, Footer, Global Settings, SEO sederhana, GA4, preview full page, draft/save/publish, snapshot homepage, dan restore versi lama. Semua animasi harus premium namun tetap aman untuk mobile dan low-spec, dengan maksimal satu parallax utama dan fallback otomatis untuk perangkat lemah.
